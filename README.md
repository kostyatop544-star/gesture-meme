# gesture_meme — веб-версия

Открывается прямо по ссылке в Safari/Chrome на телефоне, без установки.
Камера + распознавание лица/рук работает через MediaPipe Tasks Vision
(JS-версия той же технологии, что была в оригинальном тиктоке), библиотека
мемов сохраняется прямо в браузере устройства.

## Стартовый набор реакций

При первом открытии библиотека не пустая — туда подгружены **10 оригинальных
реакций** (не копии существующих мемов, а простая флэт-графика, нарисованная
с нуля): шок / mind blown, истерический смех, deadpan, скептичный взгляд
искоса, facepalm, sus, вымученная улыбка, confused, крик, язык наружу.

Категории выбраны по итогам разбора самых живучих форматов реакций
2020–2026: экспрессивные крупные планы лица без слов ("silent review"),
side-eye/скептицизм, facepalm, deadpan-юмор — это устойчиво повторяющиеся
паттерны в трендах TikTok/Reels все эти годы, в отличие от конкретных
персонажей-мемов (Sonic.exe, distracted boyfriend и т.п.), которые быстро
устаревают и к тому же защищены авторским правом — поэтому их сюда
не встраивали.

Векторы у стартовых реакций — приблизительные (подобраны вручную под
типичную мимику этой категории), не записаны с реального лица. Это рабочая
отправная точка, но точность будет заметно выше, если ты:
1. Откроешь библиотеку → удалишь стартовую реакцию (или оставишь)
2. Добавишь свою — с реальной картинкой и **записанной с себя позой**

Файл стартового набора — `seed_memes.js`, можно смело редактировать/удалять
записи оттуда или дописывать свои по тому же формату.

## Файлы

- `index.html` — разметка
- `style.css` — стили (тема "редактор кода", в духе оригинального видео)
- `app.js` — вся логика: камера, детекция, подбор мема, хранение
- `seed_memes.js` — стартовый набор из 10 оригинальных реакций

Никакой сборки не требуется — это чистый HTML/JS, работает как есть.

## Как выложить на GitHub Pages (5 минут, без токена, руками через сайт)

1. Зайди на **github.com**, войди в свой аккаунт.
2. Нажми **New repository** (зелёная кнопка). Назови, например `gesture-meme`.
   Сделай его **Public** (для бесплатного GitHub Pages это обязательно,
   если у тебя не платный аккаунт с приватными Pages).
3. Не добавляй README при создании — оставь репозиторий пустым.
4. На странице пустого репозитория нажми **uploading an existing file**.
5. Перетащи туда все три файла: `index.html`, `style.css`, `app.js`.
6. Нажми **Commit changes**.
7. Перейди в **Settings → Pages** (в левом меню репозитория).
8. В разделе **Build and deployment → Source** выбери **Deploy from a branch**.
9. Branch: **main**, папка: **/ (root)** → **Save**.
10. Подожди 1-2 минуты, обнови страницу — вверху появится ссылка вида:

```
https://ТВОЙ_НИК.github.io/gesture-meme/
```

Это и есть ссылка, которую можно открыть в Safari или Chrome на телефоне.

## Важно про HTTPS и камеру

Браузеры дают доступ к камере **только по https** (или на localhost).
GitHub Pages отдаёт сайт именно по https "из коробки" — так что всё
сработает сразу, ничего дополнительно настраивать не нужно.

## Как пользоваться

1. Открой ссылку на телефоне, разреши доступ к камере.
2. Нажми **«+ добавить мем»**.
3. Выбери картинку мема из галереи, впиши название.
4. Встань перед камерой в нужной позе → **«записать текущую позу»** →
   **«сохранить мем»**.
5. Повтори для нескольких мемов.
6. Вернись на главный экран — при повторении записанной позы сверху
   появится соответствующая картинка.

Библиотека хранится в `localStorage` браузера **на конкретном устройстве**.
Если открыть ссылку на другом телефоне — библиотека будет пустой, там
нужно будет добавить мемы заново (это ограничение локального хранения,
не серверного).

## Технические детали

- **Лицо**: `FaceLandmarker` из MediaPipe, включая готовые blendshapes
  (`jawOpen`, `mouthSmileLeft/Right`, `browInnerUp`, `eyeBlinkLeft/Right`,
  `tongueOut` — да, высунутый язык, как в примере с Соником, здесь
  ловится "из коробки", в отличие от нативной Swift-версии).
- **Руки**: `HandLandmarker`, до 2 рук, положение запястья относительно
  bounding box лица.
- **Сопоставление**: евклидово расстояние между текущим вектором признаков
  и эталонными векторами сохранённых мемов, с усреднением по последним
  6 кадрам, чтобы картинка не "дёргалась".
- **Модели** грузятся с CDN Google при первом заходе (несколько МБ),
  дальше кешируются браузером.

## Что можно улучшить дальше

- Перенести библиотеку на бэкенд (Firebase/Supabase), чтобы делиться
  мемами между устройствами и людьми.
- Добавить экспорт/импорт библиотеки в JSON-файл (кнопка "скачать" /
  "загрузить") — простое решение без бэкенда, чтобы переносить между
  устройствами вручную.
- PWA-манифест + service worker, чтобы можно было "добавить на экран
  домой" и открывать как обычное приложение, без адресной строки.
